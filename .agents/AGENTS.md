# Project Custom Rules

## Pull Request Strategy
- **Small and Targeted PRs:** Always aim to create smaller, focused Pull Requests rather than large, sweeping ones.
- **Proactive PR Suggestion:** When a distinct, logical unit of work (a feature, a bug fix, or a set of related tests) is completed and verified, proactively suggest committing the changes and opening a Pull Request before moving on to unrelated tasks.
- **Ideal Merge Text:** When suggesting a PR, automatically generate a well-structured PR title and description for the user to copy-paste.
